Your computer is composed of four classical functions: Input, Process, Storage, and Output. Figure 1 summarizes these functions graphically. Let's break it down.

Input

Data that needs to be processed or stored is encoded to enter the system. This process involves converting information from analog to digital format so that the computer can effectively process it. Various devices such as the keyboard, mouse, and microphone are commonly used to input data into the system. Some examples of data input situations include uploading files, typing notes, and recording audio.

Process

Once data is collected, it undergoes various operations to be transformed into useful information. This process is crucial as it allows raw data to be interpreted and utilized effectively. Common examples of how data is processed include searching for specific information within a database, performing calculations to manipulate numerical data, analyzing patterns or trends to gain insights, and converting data from one format to another to ensure compatibility with different systems or applications. These processes are essential to derive meaningful insights and make informed decisions based on the data at hand.

Storage

Processed data is typically stored for future use to ensure it can be accessed or utilized as needed. In the short term, a computer keeps this information in its primary storage, known as RAM, which allows quick access for ongoing tasks. For long-term storage, information is saved on your computer’s solid-state drive (SSD) or hard disk drive (HDD). Recently, Cloud Storage has become a popular option, allowing data to be stored online and accessed from most devices with internet connectivity. For offline and portable use, information can be saved on flash drives, providing a convenient way to transfer data between devices.

Output

After processing, data is presented in formats that align with the user’s specific goals, making it easier to understand and use. Various devices are utilized to display information, including the monitor, printer, and speaker. These devices allow the user to interact with the processed data in different ways. For instance, playing music files through a speaker provides an audio output, while printing word documents using a printer offers a physical copy of the information. Additionally, slideshow presentations displayed on a monitor visually present data to an audience, enhancing communication and comprehension.

Definition

Software development is the process of conceptualizing, designing, programming, and deploying software applications that address particular needs or solve specific problems. It involves transforming ideas or requirements into functional software through a series of steps, including coding, testing, and debugging to ensure the software operates effectively. The goal of software development is to build reliable, efficient, and user-friendly solutions tailored to various business or individual needs. Central to this process is the structured approach to managing development, often referred to as the Software Development Life Cycle (SDLC), which outlines the stages involved from idea to deployment.

Software Development Life Cycle

The figure on the left outlines the Software Development Life Cycle. It begins with Requirement Analysis, where the requirements of the software are defined in detail, and how these requirements will be implemented. This is followed by System Design, where the software is designed through methods such as flowcharts and pseudocode. In Implementation, a working program is made. Testing is essentially checking if the program satisfies the requirements. A way of viewing it is messing around with the program in an attempt to break it, and fixing these. In Deployment, the program is released for the end users. This is done when the program is finalized with no more issues that can be identified. Finally, Maintenance encompasses the upkeep of the program by updating to add new features and modifications, and fixing flaws identified by users.

Example

Let's try using the SLDC to create a program to solve something simple. Say we want to make a random number guessing game. For the first step, we identify the goal and requirements of the software:


Goal: To create a game where the computer selects a random number within a given range, and the user has to guess the number.

Requirements:

  • The program generates a random number from 1 to 100
  • The user enters guesses that have to be integers from 1 to 100 until the correct number is found
  • The program provides feedback on whether or not the guess was too high, too low, or correct
  • The program keeps track of the number of guesses made

The second step involves outlining how the code would look like by using something like pseudocode:

  1. Generate a random number between 1 and 100.
  2. Initialize attempts to 0.
  3. Repeat until the user guesses the correct number:
    1. Ask user for an integer input.
    2. If input is not a valid integer, ask again.
    3. Increase attempts by 1.
    4. If input is less than the random number, print "Too low."
    5. If input is greater than the random number, print "Too high."
    6. If input is equal to the random number, print "Congratulations!" and the number of attempts.

For steps 2 and 3, this program can be coded on a variety of languages such as Python, C++, and Java.


Meanwhile, the final two steps can be done using tools such as GitHub, which allow you to publish and release new versions of your code.

Comparison Operators

The next type of operator is comparison operators, which is very similar to actual comparison operators in real mathematics. It returns a boolean True or False based on the result of the expression.


# A thing worth noting is that these operators is not limited to being used on floats or integers, they can actually also be used in things such as strings or lists

# == compares if the first value is equal to the other
print(5 == 10) # outputs False
print(5 == 5) # outputs True
print("peppa" == "peppa") # outputs True

# != compares if the first value is NOT equal to the other
print(5 != 10) # outputs True
print(5 != 5) # outputs False
print("peppa" != "george") # outputs True

# < and >, compares if the first value is less than or greater than the second value, respectively.
print(5 < 10) # outputs True
print(5 > 10) # outputs False
print(5 > 5) # outputs False

# For strings, python does a comparison for each character and moves onto the next one if the previous are the same. Then, it compares the unicode values of the characters
print("peppa" < "george") # outputs False, since p and g (the first elements of each string) are different, and g = 103, while p = 112
print("jorge" > "jorj") # outputs False, since the first differing elements of the strings is g and j, and g = 103 and j = 106
print("jorj" < "jorje") # outputs True, since Python is going to hit a critical point where the first value has no more characters, and the second still has e. In this case, the longer string is considered greater
# It's crucial to note that since it refers to the unicode values of the characters, it is case sensitive since the lowercase and uppercase of a letter havve different unicode values

# For lists, it also does a lexicographic comparison similar to strings. It looks for the first unique values and compares them.
print([1,2,3,4] < [2,4,6]) # outputs True

# <= and >=, compares if the first value is less than/equal to or greater than/equal to the second value, respectively. It's basically the same as < and > with the extra =
print(5 <= 5) # outputs True
print("jarvis" >= "jarvis") # outputs True

Bitwise Operators

Finally, bitwise operators are used to do bitwise calculations on integers. The integers are first converted to binary, and the desired operations are done on it.


# In Python, the bitwise operators are AND (&), OR (|), XOR (^), NOT (~), right shift (>>), and left shift (<<)

# &, |, and ^ can all be visualized this way:
'''
Suppose you have two integers, 10 and 7. Their binary representations are as follows:
1010 -> 10
111  -> 7
Since their lengths are not the same, we need to put leading zero/s to the shorter one until they are. Take note that this won't happen if the binary representations are the same length already
1010
0111

For &, it checks through each position and returns a 1 if and only if both values are 1. Thus,
1010
0111
----
0010
10 is the resulting binary representation, which is 2. So, 10 & 7 is equal to 2.

For |, it checks through each position and returns a 1 if at least one of the values is 1. Thus,
1010
0111
----
1111
1111 is the resulting binary representation, which is 15. So, 10 | 7 is equal to 15.

Finally, for ^, it checks through each position and returns a 1 if and only if one of the values is 1.
1010
0111
----
1101
1101 is the resulting binary representation, which is 13. So, 10 ^ 7 = 13.
'''
# For these three operations, the order of the operands does not matter.

# As you may have guessed by now, ~ just gets the binary representation of a number, flips the 1s and 0s, and returns its integer form, but the catch it is evaluates the result using two's complement
print(~7) # this outputs -8
# In a nutshell, using ~ on an integer just returns -(n+1), where n is the integer

# >> and << shift the binary representation to the right or to the left by a number of units, respectively.
print(10 >> 2) # this outputs 2, since 1010 shifted 2 units to the right is just 10, whose decimal representation is 2
print(10 << 2) # this outputs 40, since 1010 shifted 2 units to the left is 101000, whose decimal representation is 40

Conventions

Here's a list of key Python conventions that you should follow:

  • Use snake_case for variable and function names: Use lowercase words separated by underscores (e.g., my_variable, calculate_total()).
  • Use ALL_CAPS for constants: Constants should be written in all uppercase letters with underscores separating words (e.g., MAX_SIZE).
  • Indentation: In Python, indentation is crucial for defining code structure, as it replaces braces `{}` used in other languages. Improper indentation will result in errors, making it essential for code execution.
  • Use descriptive variable names: Variables should have meaningful names that describe their purpose (e.g., user_age instead of x).
  • Use blank lines to separate code sections: Use one or two blank lines to separate logical sections of code (e.g., between functions or class definitions).
  • Comment your code: Use comments to explain complex logic and document your code with # for single-line comments.
  • Docstrings for functions and classes: Use triple quotes (""") to document functions, methods, and classes. These are called docstrings.
  • Use spaces around operators: For readability, use spaces around operators (e.g., total = a + b instead of total=a+b).
  • Import modules at the top of the file: All imports should be at the beginning of the file.

Four Pillars of Object Oriented Programming (OOP)

Before we get into actual modular programming, let's first talk about the four pillars of OOP: Abstraction, Encapsulation, Inheritance, and Polymorphism.


Typically, the four pillars are visualized as literal pillars as shown in Figure 1. Let's break down what each of these pillars mean in Python.


Abstraction focuses on exposing only the necessary details and hiding the others from the user. It allows you to define a blueprint or interface without getting bogged down by implementation details. In Python, abstraction is achieved through classes and methods, allowing you to model real-world entities with only relevant properties and behaviors. Another example of abstraction is Python libraries, since the user is only told how to use the library, and not necessarily what happens behind the scenes.


Encapsulation is the concept of bundling data and methods that operate on that data within a single unit, or object. In Python, this is done using classes. Encapsulation essentially wraps the data in a "capsule", such that they cannot be accessed or modified outside of it (unless there are getters and setters, which we'll discuss later on). For example, you can use private variables (by prefixing them with an underscore) to restrict access from outside the class, allowing you to define a clear interface for interacting with the object.


Inheritance is a mechanism that allows one class (called a child or subclass) to inherit properties and behaviors from another class (called a parent or superclass). This promotes code reuse and helps create a hierarchical relationship between classes. For instance, you can have a parent Gadget class, with specific subclasses such as Phone or Laptop that can inherit common properties and methods from Gadget, while also having the option to have unique behaviors.


Polymorphism allows different classes to be treated as instances of the same class through a common interface. It enables a single function or method to operate on different types of objects. In Python, polymorphism is often used when a parent class method is overridden in a child class, providing specific behavior for the child while maintaining a consistent interface. This flexibility allows you to use a unified approach to handle different data types and objects, making the code more flexible.

Importing Libraries

Libraries, also known as modules, are essentially files composed of functions (and sometimes, variables) designed to make your life easier. In order to include a library in our code, we use the import keyword.

To demonstrate this, let's compare the implementation of a factorial function on base Python, versus Python with the math library.


n = int(input())
result = 1
for i in range(1, n + 1):
    result *= i
print(result)
                            

import math
n = int(input())
print(math.factorial(n))

As you can see, using the math module to get the factorial is much easier and shorter. There are other libraries available aside from math, which can even be complex stuff such as machine learning.

It's also worth noting that you can import multiple libraries on a single program. You are not limited to just one library.

Functions

A function in Python is a block of reusable code designed to perform a specific task. It allows you to organize your code into logical chunks, making it easier to read, test, and maintain. You define a function using the def keyword, followed by the function name and parentheses (), which may contain parameters. A function can return a result using the return statement, or simply perform an action.


To apply this, let's try to turn our previous square root example into a function.


def factorial(n):
result = 1

In this example, our factorial code now becomes reusable. Instead of having to write the same block of code every time we need it, we can just call factorial(n) and get its value.

Objects

In Python, everything is an object, which means functions, variables, data structures, and even classes are all objects. Object-Oriented Programming (OOP) is a paradigm that uses objects and classes to structure your code, allowing you to model real-world entities in your programs.


Defining a Class

A class is a blueprint for creating objects. Each object created from the class can have its own properties (attributes) and behaviors (methods). Attributes are variables that belong to an object, while methods are functions that belong to the class. You can define multiple attributes and methods within a class.


# Let's make a simple Car class:
class Car: # It's important to note that classes in Python follow CamelCase
    def __init__(self, brand, model): # In other coding languages, this is essentially a constructor
        self.brand = brand  # Attribute: car's brand
        self.model = model  # Attribute: car's model

    def display_info(self): # This is a method
        print(f"This car is a {self.brand} {self.model}")

# Creating an object (an instance of the Car class)
my_car = Car("Toyota", "Corolla")
my_car.display_info()  # outputs "This car is a Toyota Corolla"

Application - Encapsulation


# Encapsulation can be seen in Python with the usage of __ at the beginning of a variable to essentially declare it as "private"
# Let's use this example of a BankAccount class. In it, balance has __ at the beginning. This means that without a function to get the balance (get_balance), there's no way for stuff outside of the class to get the value of __balance
class BankAccount:
    def __init__(self, owner, balance):
        self.owner = owner
        self.__balance = balance  # Private attribute

    def deposit(self, amount):
        self.__balance += amount

    def get_balance(self):
        return self.__balance

account = BankAccount("Alice", 1000)
account.deposit(500)
print(account.get_balance())  # outputs 1500

Application - Inheritance


# Inheritance is evident in Python as you can have parent classes and subclasses that share attributes and possibly methods with its parent.

class Animal:

Application of Objects

To apply concepts on objects, let's try making a simple program that stores various manhwa (Korean comics), as well as the current progress of the reader by chapter.


# A dictionary to store the manhwas
manhwas = {}

class Manhwa:
    def __init__(self, title, genre, personal_rating, chapter = 0): # if there is no specified chapter, the default is 0, meaning the person hasn't started reading but has put it on their list
        self.title = title
        self.genre = genre
        self.rating = personal_rating
        self.chapter = chapter
        manhwas[title] = self

    def print_details(self):
        print(f"{self.title}".ljust(40) + f"({self.genre})".ljust(12) + f"{self.rating} | Chapter {self.chapter}") # using f strings to make life easier

    def read_chapter(self): # for when the user has just read a chapter
        self.chapter += 1

    def set_chapter(self, chapter): # for when the user wants to set the chapter to a specific spot
        self.chapter = chapter

    def change_rating(self, rating): # for when the user changes their mind about how good a manhwa is
        self.rating = rating

# function to print all the manhwas in the dictionary
def print_all_manhwas():
    print("Current Manhwa List:")
    for manhwa in manhwas.values(): # using a for loop to iterate through the manhwas
        manhwa.print_details()
    print()

manhwa1 = Manhwa("Solo Leveling", "Action", 9.5)
manhwa2 = Manhwa("Tower of God", "Adventure", 9.0)
manhwa3 = Manhwa("Legend of the Northern Blade", "Action", 8.5, 25)

# Print the details of all the manhwas
print_all_manhwas()

# Let's try to do some actions:
print("Reading a chapter of Solo Leveling...")
manhwa1.read_chapter()
manhwa1.print_details()

print("\nSetting Tower of God to chapter 100...")
manhwa2.set_chapter(100)
manhwa2.print_details()

print("\nChanging Legend of the Northern Blade's rating to 9.0...")
manhwa3.change_rating(9.0)
manhwa3.print_details()
print()

# Print the updated list of all manhwas
print_all_manhwas()

Exception Handling

Exceptions take place when the syntax of a program is correct, but the execution of the code causes an error or still runs but produces a wildly inaccurate result.


a = float(input())
print("the reciprocal of the number you input is:")
print(1 / a)

In the above example, If the input is 0, then the result will be a ZeroDivisionError, as dividing a number by 0 raises an error in Python. Conversely, if you input anything other than a number (for example: “f1sh”), then a ValueError will be raised, as the float() method requires a number, not a string (even if it has numbers). There are several ways to handle exceptions, one of which is to avoid exceptions altogether by checking if the values are valid.


However, a better example would be to use a try-catch block, which allows you to catch several cases of errors in one go.

How it functions is that within the try block, the code will be attempted to be executed without error. In the event of an error occurring, the sequence will jump to the code within the except block. You can catch several exceptions with one block, or allocate individual exceptions to a different exception block, as shown below:


try:
    a = float(input())
    print("The reciprocal of the number you input is:")
    print(1 / a)
except ValueError:
    print("Input a number!")
except ZeroDivisionError:
    print("Zero has no defined reciprocal!")
except: 
    print("Something went wrong, and we don't know what happened!")
finally:
    print("Thanks for using this program, broski.")

It's important to note that the block will only count the first except that it finds that fits the exception. Then, if a finally block is present, it gets executed regardless of whether or not an exception was found.


Another thing to note is that exceptions are a class, allowing its features to be used and inherited. This implies that it's also possible to make your own exceptions by making a class that extends Exception.


class SecretError(Exception):
    pass
    
try:
    password = str(input("What's the password big bro? "))
    if len(password) == 8:
        raise SecretError("You found a secret!")
    else:
        print("That's not a secret password.") 
except SecretError as e:
    print(f"Secret Found: {e}")
finally:
    print("Thanks for trying to guess a secret password!")

Slicing

Slicing allows you to access a subset of a list, string, or other sequence types by specifying a range (start, stop) and an optional step. Let's apply this given a sample array:

When slicing, it's important to note that given a start and stop index, the resulting subset includes the element at the start index, but not the element at the stop index. Another thing is that the first element is at index 0, not 1.


# The general syntax is sequence[start:stop:step]
cars = ['sedan', 'truck', 'suv', 'coupe', 'convertible', 'sports', 'van', 'wagon', 'hatchback']

print(cars[1:4]) # ['truck', 'suv', 'coupe']

# If the stop is not specified, it assumes that it's up until the end
print(cars[1:]) # ['truck', 'suv', 'coupe', 'convertible', 'sports', 'van', 'wagon', 'hatchback']

# If the start is not specified, it assumes that the start index is 0
print(cars[:5]) # ['sedan', 'truck', 'suv', 'coupe', 'convertible']

#If neither is specified, then
cars[::] = cars

#If start = stop, then
cars[2:2] = []

# Adding a step means that it includes the first element, then every nth element after, with n being the step. A step of 1 is the default.
print(cars(::2)) # ['sedan', 'suv', 'convertible', 'van', 'hatchback']

# Negative values is where things get interesting. As a guide, the last value of a sequence is at index -1, the second last at index -2, and so on.
# Assuming that the step is positive, the start HAS TO BE further to the left than the stop, or else it will return an empty sequence.
# But if the step is negative, that rule flips.

print(cars[4:-5]) # []
# This actually returns empty, since the indices 4 and -5 refer to the exact same index in this case.

print(cars[4::-1]) # ['convertible', 'coupe', 'suv', 'truck', 'sedan']
# In here, if the stop is unspecified with a negative step, it assumes that the values are from index 4 leftwards.

print(cars[:2:-1]) ['hatchback', 'wagon', 'van', 'sports', 'convertible', 'coupe']
# Conversely, if the start is unspecified with a negative step, it assumes that the values are from the rightmost element to before index 2 (which is index 3).

print(cars[4:2:-1]) # ['convertible', 'coupe']
# Here, it starts at index 4, then goes leftwards until before the element at index 2

print(cars[::-1]) # ['hatchback', 'wagon', 'van', 'sports', 'convertible', 'coupe', 'suv', 'truck', 'sedan']
# This is actually a quick way to reverse the order of elements in a sequence

File I/O

In Python, File I/O (Input/Output) refers to the reading and writing of files. It's essential when you want your programs to persistently save data, or retrieve data from a file for processing. Python provides several built-in functions for handling file I/O, such as open(), read(), write(), and close(). Let's explore the basics.

Opening a File

To interact with a file, you first need to open it. Python’s open() function allows you to open a file in different modes:

  • 'r': Opens the file for reading (default mode).
  • 'w': Opens the file for writing (overwrites the file if it already exists).
  • 'a': Opens the file for appending (adds data to the end of the file).
  • 'b': Opens the file in binary mode (useful for non-text files like images).

# Example of opening a file in read mode
file = open("example.txt", "r")
print(file.read())  # This reads and prints the content of the file
file.close()  # Always close the file after finishing

Reading from a File

Once a file is opened, you can use various methods to read its contents:

  • read(): Reads the entire file content.
  • readline(): Reads one line at a time.
  • readlines(): Returns a list of all the lines in the file.

file = open("example.txt", "r")

# Reading all the content
content = file.read()
print(content)

# Reading line by line
file.seek(0)  # Reset file pointer to the beginning
for line in file:
print(line.strip())  # Prints each line

file.close()

Writing to a File

To write to a file, you can open it in 'w' or 'a' mode. Remember that opening a file in write mode will overwrite the entire file.


file = open("example.txt", "w")  # Open the file in write mode
file.write("Hello, World!\n")  # Write a line to the file
file.write("This is another line.\n")
file.close()  # Close the file

# Appending to the file
file = open("example.txt", "a")
file.write("Appending a new line!\n")
file.close()

Using "with" for File Handling

Python provides a better way to handle files using the with statement. It ensures the file is automatically closed after the block is executed, even if exceptions occur.


with open("example.txt", "r") as file:
    content = file.read()
    print(content)  # File is automatically closed after this block
